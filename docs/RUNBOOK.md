# RagAnalyzer: запуск и проверка

## 1. Какая проблема есть

Проекту нужны PostgreSQL с pgvector и локальная Ollama, но в репозитории нет пошаговой инструкции запуска. Без нее легко запустить только часть системы и получить ошибки на индексации или чате.

## 2. Как исправлять

Перед запуском нужно поднять инфраструктуру, проверить модели Ollama, затем стартовать Spring Boot приложение и открыть статический UI.

## 3. Команды запуска

Поднять PostgreSQL:

```powershell
docker compose up -d postgres
```

Проверить, что Ollama работает:

```powershell
ollama list
```

Скачать модели, если их нет:

```powershell
ollama pull nomic-embed-text:latest
ollama pull qwen2.5:7b
```

Запустить приложение:

```powershell
mvn spring-boot:run
```

Открыть UI:

```text
http://localhost:8080
```

## Проверка работоспособности

Минимальный ручной сценарий:

1. Открыть `http://localhost:8080`.
2. Загрузить `.txt`, `.pdf` или `.docx`.
3. Дождаться статуса `INDEXED`.
4. Выбрать документ.
5. Задать вопрос по содержимому.

Если загрузка падает:

- проверить, запущен ли PostgreSQL;
- проверить, применились ли Flyway-миграции;
- проверить, отвечает ли Ollama на `http://localhost:11434`;
- проверить, совпадает ли размерность эмбеддингов с `vector(768)`.

## Полезные проверки для разработки

Компиляция без тестов:

```powershell
mvn -DskipTests compile
```

Тестовая компиляция:

```powershell
mvn test-compile
```

Полный тестовый прогон, когда появятся тесты:

```powershell
mvn test
```
