# RagAnalyzer: известные проблемы и улучшения

## 1. Какая проблема есть

Во время первичного изучения проекта видны несколько мест, которые могут мешать запуску, сопровождению или стабильной работе RAG-пайплайна.

## 2. Как исправлять

Исправлять лучше небольшими шагами: сначала конфигурация запуска, затем кодировка и читаемость, затем надежность индексации и тесты.

## 3. Найденные проблемы

### `docker-compose.yml` смешивает Compose и application config

В файле после секции `volumes` находится:

```yaml
ollama:
  base-url: http://localhost:11434
  embedding-model: nomic-embed-text:latest
```

Это похоже на настройки Spring Boot, но они лежат не в `application.yml`. Для Docker Compose это не service и не валидная top-level секция.

Как исправить:

- удалить этот блок из `docker-compose.yml`;
- оставить настройки Ollama в `src/main/resources/application.yml`;
- если нужен Ollama-контейнер, добавить его как `services.ollama`.

### Mojibake в русских строках и комментариях

Во многих Java/XML/JS файлах русские строки отображаются как `РќР°...`. Это мешает читать код, сообщения ошибок и prompt для LLM.

Как исправить:

- определить фактическую кодировку файлов;
- аккуратно перекодировать исходники в UTF-8;
- после перекодировки обязательно прогнать компиляцию и ручной smoke-тест.

### Нет автоматических тестов

В `pom.xml` есть `spring-boot-starter-test`, но тестовые файлы не обнаружены.

Как исправить:

- добавить unit-тесты для `ChunkService`, `LlmService.cleanAnswer`, `VectorStoreService.toPgVector`;
- добавить интеграционный тест для контроллеров с mocked services;
- позже добавить Testcontainers для PostgreSQL + pgvector.

### Индексация документа выполняется синхронно

`DocumentService.uploadDocument` сразу извлекает текст, режет на чанки, вызывает Ollama и пишет embeddings. Большой документ или медленная Ollama могут надолго занять HTTP-запрос.

Как исправить:

- сохранять документ со статусом `UPLOADED` или `PROCESSING`;
- запускать индексацию асинхронно;
- отдавать клиенту статус и обновлять его через polling или SSE.

### Ошибки извлечения изображений только пишутся в stderr

Если извлечение изображений падает после успешной индексации, приложение печатает предупреждение через `System.err.println`.

Как исправить:

- заменить на нормальный logger;
- сохранять диагностическую информацию отдельно от статуса индексации;
- показать пользователю, что текст проиндексирован, но изображения недоступны.
